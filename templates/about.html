Project Technical Specifications

01 Introduction

The Public News Platform is a centralized intelligence hub designed to solve the problem of information overload. Its primary objective is to provide professional-grade, real-time news synthesis through a streamlined, personalized interface. Importance lies in its ability to filter global noise into actionable data for decision-makers.

02 Team Composition & Roles

Our team consists of 33 members (Registration IDs 193-225). Under the leadership of Member 210 (Project Lead), we utilized a hierarchical structure divided into Engineering, UX/Frontend, Quality Assurance, and Research clusters.

04 Methodology Used

We adopted the Agile Scrum methodology. This allowed for iterative development cycles, regular code reviews by the Lead Engineer, and rapid prototyping of the mobile-responsive interface based on peer feedback.

05 Software Engineering Domain

The project follows Modular System Architecture. Key principles applied include Separation of Concerns (decoupling logic from presentation), Fault Tolerance (API fallback mirrors), and Input Sanitization for secure data processing.

06 Technologies & Tools

Python 3.x Flask (Framework) SQLite3 (DBMS) Tailwind CSS NewsAPI Integration GitHub (Version Control) Vercel (Cloud Deployment)

07 Database Management System

We selected SQLite3 for its zero-configuration, serverless nature, which is ideal for high-speed read/write operations in a news environment. The relational schema tracks executive briefing history, ensuring data persistence and session auditability.

08 Implementation Overview

The system utilizes a Request-Response Architecture. When a user requests a briefing, the Flask backend asynchronously fetches data from global news servers, processes the JSON payloads, and renders a dynamic, print-ready executive summary via Jinja2 templating.
